# Steam 출시 전 추천 추가 기능 (TIME-LOOP)

현재 코드베이스를 기준으로, Steam 상용 출시 품질을 위해 우선순위대로 제안합니다.

## 1) 게임성/콘텐츠 (가성비 최고)

### 1-1. 수동 난이도(4단계) + 별도 리더보드
- 현재는 싱글/엔들리스와 루프 기반 속도 증가만 존재.
- Easy / Normal / Hard / Overclock 같은 고정 난이도 프리셋을 추가하면 접근성이 크게 개선됨.
- 난이도별로 하이스코어/콤보를 분리 저장하면 재도전 동기가 생김.

### 1-2. 정식 패턴 차트(랜덤 생성 보완)
- 현재 노트 생성은 BPM 기반 랜덤 스폰 비중이 높음.
- 공식 곡 2~5곡은 JSON 차트(타이밍, 레인, 노트 타입)로 고정 패턴을 넣으면
  "실력 게임" 완성도가 확 올라감.
- 랜덤은 Endless 전용으로 유지 추천.

### 1-3. 튜토리얼 2단계화
- 지금 도움말은 잘 되어 있으나 실제 플레이 튜토리얼(인터랙티브)은 없음.
- Step 1: 기본 노트/판정
- Step 2: 루프 이후 고스트(PARADOX) 회피
- 초반 이탈률 감소에 매우 효과적.

## 2) Steam 필수/준필수 기능

### 2-1. Steamworks 업적/통계 연동
- 현재 업적 시스템은 localStorage 기반으로 구현되어 있어 구조가 좋음.
- unlockAchievement()를 Steam API 호출로 브릿지하면 빠르게 이관 가능.
- 플레이타임, 총 루프, 최고 정확도 같은 Stats 추가 추천.

### 2-2. Steam Cloud 세이브
- 현재 설정/업적/점수는 브라우저 저장소 기반.
- Steam Cloud 연동으로 PC 변경 시 데이터 유지.
- 유저 만족도 체감이 큼.

### 2-3. 리더보드
- 최소 단위:
  - 공식곡 싱글 점수
  - 엔들리스 최고 루프
- 점수 산식/판정 규칙이 명확해질수록 공정성 인식이 올라감.

## 3) UX/품질 (리뷰 점수에 직접 영향)

### 3-1. 오디오 지연 보정(Offset 캘리브레이션)
- 리듬게임 핵심: 사용자별 음향 지연 보정.
- Global Offset(ms) 슬라이더와 테스트 비트를 제공.
- 지금 judgeWindow가 고정이라 기기별 불리함 가능.

### 3-2. 입력 옵션 확장
- 현재 4키 리바인딩은 이미 있음.
- 다음을 추가 추천:
  - 양손 대체 키 프리셋
  - 동시에 같은 키 중복 입력 허용 옵션
  - 키다운/키업 판정 모드 선택

### 3-3. 결과 화면 확장
- 현재 점수/정확도/콤보/랭크는 훌륭함.
- 여기에 판정 통계(Perfect/Miss/Paradox), 구간별 정확도, 재시도 버튼(즉시 리트라이) 추가.

## 4) 밸런스/시스템

### 4-1. 점수 산식 정규화
- 현재 콤보 가산 방식이 단순해서 인플레이션 가능.
- 곡 길이/BPM 차이를 보정하는 정규화 점수(예: 1,000,000 cap) 고려.

### 4-2. 체력 시스템 시각 피드백 강화
- Endless 체력은 구현되어 있음.
- 저체력 구간(Clutch)에서
  - UI 경고
  - SFX
  - 배경 효과 단계적 변경
  추가하면 긴장감 상승.

### 4-3. 고스트 노트 가독성 옵션
- 색약/가독성 옵션으로 고스트 외곽선 두께, 색상 프리셋 제공.

## 5) 커스텀 곡 안정성

### 5-1. 파일 경로/보안 정리
- preload에서 songs 폴더 스캔 구조는 좋지만,
  메인 프로세스와 렌더러의 접근 방식이 혼재됨.
- 로컬 파일 접근은 preload IPC 단일 경로로 정리 권장.

### 5-2. 메타데이터 분리
- 곡명/BPM/표준 offset/아트워크를 JSON 메타로 분리하면
  커스텀 생태계 품질이 올라감.

### 5-3. 곡 검증 파이프라인
- 지원 포맷, 길이, 샘플레이트, 디코딩 실패 처리 메시지를 표준화.

## 6) 출시 운영

### 6-1. 난이도별 도전과제(주간 미션)
- "오늘의 루프", "Paradox 0회 클리어" 등 주간 미션.
- 리텐션 확보에 효과적.

### 6-2. 데모 버전
- Steam Next Fest용 데모로
  - 공식곡 1개
  - Endless 제한 루프
  - 업적 일부 비활성
  구성 추천.

### 6-3. 텔레메트리(익명)
- 튜토리얼 이탈 지점, 평균 실패 루프, 키 설정 변경률 수집.
- 밸런스 패치 근거 확보.

## 추천 구현 순서 (8주 예시)
1. 오프셋 캘리브레이션 + 결과 상세 통계 + 즉시 재시작
2. 난이도 4단계 + 정식 차트(공식곡)
3. Steam 업적/통계 + Cloud
4. 리더보드 + 커스텀 곡 검증 개선
5. 데모 빌드/스토어 페이지 자산 정리

## 핵심 한 줄
- 이 게임의 차별점은 "루프 + 고스트 패러독스"이므로,
  Steam 출시에서는 "공정성(판정/오프셋)"과 "반복 동기(리더보드/업적/난이도)"를 먼저 강화하는 것이 가장 효율적입니다.
